# effective-mobile-case

Тестовое задание на позицию Junior System Analyst.  
Кейс показывает базовые навыки системного аналитика: анализ требований, выявление противоречий, подготовку уточняющих вопросов, проектирование REST API и описание верхнеуровневой архитектуры PUSH-уведомлений.

## Кратко о задании

В рамках кейса нужно было выполнить 3 задачи:

1. Проанализировать фрагмент ТЗ по функционалу корзины интернет-магазина, найти логические противоречия, недочёты и неполные места.
2. Спроектировать REST API для мобильного экрана выбора партнёрского магазина и подготовить пример JSON-ответа по макету.
3. Описать верхнеуровневую архитектуру отправки PUSH-уведомлений в мобильном приложении при микросервисной архитектуре бэкенда.

## Что сделано

- выявлены противоречия и слабые места в исходном ТЗ;
- подготовлена исправленная версия требований по корзине;
- составлены уточняющие вопросы к PM и бизнес-заказчику;
- спроектирован REST API endpoint для загрузки экрана партнёрских магазинов;
- подготовлен пример JSON-ответа с данными для мобильного клиента;
- описана архитектура PUSH-уведомлений: источники событий, Event Bus, Notification Service, Token Storage, шаблоны, провайдеры FCM/APNs и мониторинг.
